# MultiMeet — Specify (기능 명세서)

> 시스템이 구현해야 할 기능의 상세 명세

---

## 1. 시스템 개요

MultiMeet는 두 가지 핵심 워크플로우를 가진다:

```
[Mode A: 회의록 모드]
음성 녹음 → 파일 업로드 → Whisper STT → Claude 번역/요약 → 한국어 회의록 → 다운로드

[Mode B: 실시간 통역 모드]  
마이크 실시간 입력 → Whisper STT (청크 단위) → Claude 번역 → 화면 즉시 표시
```

## 2. 기능 명세 (Functional Specification)

### F-01. 사용자 인증 (Authentication)

| 항목 | 내용 |
|------|------|
| ID | F-01 |
| 기능명 | 사용자 로그인/회원가입 |
| 우선순위 | P0 (필수) |

**상세 요구사항:**
- 이메일 + 비밀번호 기반 인증
- JWT 토큰 방식 (Access Token 1시간, Refresh Token 7일)
- 로그인 상태 유지 (Remember me)
- 비밀번호 찾기 (이메일 발송)

**입력:** 이메일, 비밀번호  
**출력:** JWT 토큰, 사용자 프로필  
**예외:** 잘못된 자격증명 → 에러 메시지

| 항목 | 내용 |
|------|------|
| ID | F-02 |
| 기능명 | 회의 세션 생성 및 관리 |

### F-03. Mode A: 회의록 모드 — 음성 녹음

| 항목 | 내용 |
|------|------|
| ID | F-03 |
| 기능명 | 브라우저 내 음성 녹음 |
| 우선순위 | P0 (필수) |

**상세 요구사항:**
- Web Audio API / MediaRecorder API 사용
- 녹음 컨트롤: 시작, 일시정지, 재개, 종료
- 녹음 중 시각적 피드백 (파형 또는 레벨 미터)
- 녹음 시간 표시 (HH:MM:SS)
- 지원 포맷: WebM/Opus (브라우저 기본), MP3 변환 서버사이드
- 최대 녹음 시간: 3시간
- 파일 업로드 기능 (기존 녹음 파일 처리)
  - 지원 형식: MP3, MP4, WAV, M4A, WebM
  - 최대 파일 크기: 200MB

**오프라인 처리:**
- 인터넷 연결 끊겨도 녹음 계속
- 재연결 시 자동으로 처리 재개

---

### F-04. Mode A: STT 처리 (Speech-to-Text)

| 항목 | 내용 |
|------|------|
| ID | F-04 |
| 기능명 | Whisper API를 통한 다국어 음성 인식 |
| 우선순위 | P0 (필수) |

**상세 요구사항:**
- OpenAI Whisper API 사용 (`whisper-1` 모델)
- 언어 힌트 전달 (선택한 언어 코드)
- 타임스탬프 포함 트랜스크립트 요청
- 청크 처리: 25MB 초과 시 자동 분할 처리
- 처리 진행률 표시 (예: "음성 분석 중... 45%")
- STT 결과 미리보기 및 수동 수정 가능

**결과 형식:**
```json
{
  "segments": [
    {
      "id": 1,
      "start": 0.0,
      "end": 5.2,
      "text": "Hello, let's start the meeting",
      "language": "en"
    }
  ]
}
```

```
### 1. 회의 목적

1. [안건 1]
   - 내용 요약
   - 주요 발언

### 3. 결정 사항
- [결정 내용]

### 4. Action Items
| 담당자 | 내용 | 기한 |
|--------|------|------|
| [이름] | [내용] | [날짜] |

### 5. 다음 회의
- 일정: [날짜]
- 안건: [내용]
```

**처리 방식:**
- 긴 회의록은 청크 단위로 Claude API 호출 후 통합
- 스트리밍 출력으로 생성 과정 실시간 표시
- 생성 완료 후 수동 편집 가능 (마크다운 에디터)

### F-07. Mode B: 실시간 통역 모드

| 항목 | 내용 |
|------|------|
| ID | F-07 |
| 기능명 | 실시간 음성 통역 |
| 우선순위 | P0 (필수) |

**상세 요구사항:**

**입력 처리:**
- 마이크에서 연속 오디오 스트림 수신
- 3~5초 단위 오디오 청크로 분할
- 발화 감지 (VAD: Voice Activity Detection) 적용
  - 묵음 구간에서 자연스럽게 청크 분할

**번역 파이프라인:**
```
마이크 → AudioChunk(3-5초) → Whisper STT → Claude 번역 → WebSocket → 화면 표시
```

**UI 표시:**
- 왼쪽: 원문 (영어/중국어/베트남어)
- 오른쪽: 한국어 번역
- 스크롤: 자동으로 최신 내용으로 이동
- 타임스탬프 표시
- 번역 중 로딩 인디케이터

**컨트롤:**
- 통역 시작/일시정지/재개/종료
- 번역 기록 전체 삭제
- 글자 크기 조절 (가독성)
- 전체화면 모드

**저장:**
- 세션 종료 후 번역 기록 자동 저장
- 저장된 기록으로 회의록 생성 가능 (Mode A 전환)

## 3. 비기능 요구사항 (Non-functional Requirements)

### NFR-01. 성능
- 실시간 번역 응답 시간: < 3초 (청크 기준)
- 회의록 생성: 60분 분량 < 2분
- 동시 접속: 최소 10명
- 페이지 로드: < 2초

### NFR-02. 가용성
- 서비스 가용성: 99% 이상
- 점진적 오류 처리 (API 실패 시 재시도)
- 브라우저 지원: Chrome, Firefox, Safari, Edge (최신 2버전)

### NFR-03. 보안
- HTTPS 필수
- API 키 서버사이드 환경변수 관리
- 음성 파일 전송 암호화
- 회의 데이터 사용자별 격리

### NFR-04. 접근성
- 모바일 반응형 (iOS Safari, Android Chrome)
- 큰 글씨 모드 지원
- 키보드 단축키 (녹음 시작/정지: Space)

## 5. API 엔드포인트 명세

### 인증
```
POST   /api/auth/register
POST   /api/auth/login
POST   /api/auth/refresh
POST   /api/auth/logout
```

### 회의
```
GET    /api/meetings              회의 목록
POST   /api/meetings              새 회의 생성
GET    /api/meetings/:id          회의 상세
PUT    /api/meetings/:id          회의 수정
DELETE /api/meetings/:id          회의 삭제
```

### 음성 처리
```
POST   /api/meetings/:id/upload   음성 파일 업로드
POST   /api/meetings/:id/transcribe  STT 처리 요청
GET    /api/meetings/:id/transcript  트랜스크립트 조회
```

### 회의록
```
POST   /api/meetings/:id/minutes   회의록 생성
GET    /api/meetings/:id/minutes   회의록 조회
PUT    /api/meetings/:id/minutes   회의록 수정
GET    /api/meetings/:id/minutes/download?format=pdf|docx|md  다운로드
```

### 실시간 통역
```
WebSocket: /ws/interpret/:sessionId
```

### 설정
```
GET    /api/settings
PUT    /api/settings
```
