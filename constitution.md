# MultiMeet — Constitution (헌법)

> 다국어 회의 통역 & 회의록 자동화 시스템의 핵심 원칙과 가치관

---

## 1. 프로젝트 정체성 (Project Identity)

**MultiMeet**는 글로벌 영업사원이 미국(영어), 중국(중국어), 베트남(베트남어) 거래처와의 회의에서 언어 장벽 없이 소통하고, 회의 내용을 자동으로 한국어 회의록으로 정리해주는 풀스택 애플리케이션이다.

### 핵심 미션
> "언어의 차이가 비즈니스의 장벽이 되어서는 안 된다."

사용자가 회의 중 통역에 신경 쓰지 않고 **비즈니스 본질에 집중**할 수 있도록 기술이 투명하게 지원한다.

---

## 2. 핵심 가치 (Core Values)

### 2.1 실시간성 (Real-time First)
- 통역은 지연 없이 즉각적으로 제공되어야 한다
- 사용자가 "기다림"을 느끼는 순간, 서비스의 가치는 반감된다
- 스트리밍 처리를 기본 원칙으로 한다

### 2.2 정확성 (Accuracy Above All)
- 비즈니스 맥락에서 오역은 계약 손실로 이어질 수 있다
- 기술 용어, 제품명, 숫자는 특히 정확하게 처리해야 한다
- 불확실한 내용은 숨기지 않고 명확히 표시한다

### 2.3 단순함 (Radical Simplicity)
- 회의 중 복잡한 UI를 탐색할 시간은 없다
- 핵심 기능은 최소 클릭으로 접근 가능해야 한다
- 기술적 복잡성은 내부에, 사용 편의성은 외부에

### 2.4 신뢰성 (Reliability)
- 중요한 회의 중 시스템 장애는 용납되지 않는다
- 오프라인 상황에서도 기본 기능은 동작해야 한다
- 데이터는 항상 안전하게 저장되어야 한다

### 2.5 한국어 중심성 (Korean-Centric Output)
- 모든 최종 출력물은 한국어로 제공된다
- 회의록은 한국 비즈니스 문서 형식을 따른다
- 번역 결과는 자연스러운 한국어 표현을 우선한다

---

## 3. 사용자 정의 (User Definition)

### Primary User: 글로벌 영업사원
- **역할**: B2B 기술영업, 해외 거래처 관리
- **환경**: 대면 회의, 화상회의 (Zoom, Teams, Meet)
- **언어 능력**: 한국어 원어민, 외국어 기초~중급
- **니즈**:
  - 회의 내용을 놓치지 않고 싶다
  - 회의록 작성에 소요되는 시간을 줄이고 싶다
  - 상대방 말을 즉시 이해하고 싶다

### Secondary User: 팀 매니저/관리자
- 회의록 검토 및 승인
- 팀 회의 기록 아카이브 관리

## 4. 지원 언어 범위 (Language Scope)

| 언어 | 코드 | 용도 |
|------|------|------|
| 한국어 | ko | 출력 언어 (회의록, 번역 결과) |
| 영어 | en | 미국 거래처 회의 |
| 중국어 (간체) | zh | 중국 거래처 회의 |
| 베트남어 | vi | 베트남 거래처 회의 |

---

## 5. 두 가지 핵심 모드 (Dual Mode)

### Mode A: 회의록 모드 (Meeting Minutes Mode)
- 음성 녹음 → STT → 번역 → 구조화된 한국어 회의록 생성
- 회의 종료 후 문서 다운로드 (PDF, DOCX)
- 화자 구분, 타임스탬프 포함

### Mode B: 실시간 통역 모드 (Live Interpretation Mode)
- 음성 입력 → 즉각적인 한국어 번역 표시
- 스트리밍 방식으로 문장 단위 실시간 출력
- 대화형 인터페이스로 흐름 파악 가능

---

## 6. 기술 원칙 (Technical Principles)

### 6.1 AI 우선 아키텍처
- STT: OpenAI Whisper API (다국어 최고 정확도)
- 번역/요약: Claude API (claude-sonnet)
- 실시간 스트리밍: WebSocket 기반

### 6.2 풀스택 구성
- **Frontend**: React + TypeScript (Vite)
- **Backend**: Node.js + Express (TypeScript)
- **Database**: PostgreSQL (회의록 저장)
- **Storage**: 로컬 파일시스템 또는 S3 호환 (음성 파일)

### 6.3 보안 원칙
- 회의 내용은 민감한 비즈니스 정보다
- 음성 파일은 처리 후 즉시 삭제 옵션 제공
- API 키는 서버사이드에서만 관리
- 사용자 인증 필수

### 6.4 확장 가능성
- 언어 추가가 구조적으로 쉬워야 한다
- 다른 STT 엔진으로 교체 가능한 추상화 레이어
- 회의록 템플릿 커스터마이징 가능

---

## 7. 금지 사항 (Non-Negotiables)

- ❌ 번역 결과를 사용자에게 숨기거나 지연시키지 않는다
- ❌ 음성 파일을 사용자 동의 없이 외부로 전송하지 않는다
- ❌ 복잡한 설정 없이 기본 기능이 작동해야 한다
- ❌ 회의 중 인터넷 단절 시 녹음은 계속 유지되어야 한다
- ❌ 모바일 브라우저에서 핵심 기능이 동작하지 않으면 안 된다

---

## 8. 성공 기준 (Definition of Success)

| 지표 | 목표값 |
|------|--------|
| 실시간 번역 지연 | < 3초 |
| STT 정확도 (영어/중국어/베트남어) | > 90% |
| 회의록 생성 시간 (60분 회의 기준) | < 2분 |
| 사용자 만족도 | "회의에 집중할 수 있었다" |
| 모바일 지원 | iOS Safari, Android Chrome |
